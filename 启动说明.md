## 🚀 使用步骤

### 1. 访问登录页
打开浏览器访问：http://localhost:3001/

### 2. 学生端体验流程
1. 使用学生账号登录（student1 / 123456）
2. 输入成绩：
   - 填写3门必选科目成绩（Chinese Language, English Language, Mathematics Compulsory Part）
   - 至少添加2门选修科目并选择成绩
   - 成绩等级：U, 1, 2, 3, 4, 5, 5*, 5**
3. 点击"计算专业推荐"按钮
4. 查看推荐结果：
   - 根据总分匹配的专业列表
   - 成功率指示（高/中/低）
   - 学校筛选、专业筛选
   - 收藏喜欢的专业（点击星标）
5. 访问"我的收藏"查看收藏的专业
6. 在"系统设置"中修改密码

### 3. 管理员端体验流程
1. 使用管理员账号登录（admin / admin123）
2. **账号管理**：
   - 切换学生账号/管理员账号标签
   - 新增、编辑、删除账号
   - 为学生账号设置权益点数
3. **计算规则管理**：
   - 上传JSON格式的规则文件
   - 预览规则内容
   - 删除规则文件
4. **专业数据管理**：
   - 查看所有专业数据
   - 搜索专业（按学校、专业名称或ID）
   - （导入功能提示）

## 🎨 设计特色

- **简洁黑白风格**：纯黑白线条设计，简洁大方
- **响应式布局**：支持桌面、平板和手机
- **即时反馈**：操作即时响应，加载状态提示

## 📁 项目结构

```
jupas-frontend/
├── src/
│   ├── components/        # 公共组件（Button, Input, Select, Modal, Navbar）
│   ├── constants/         # 常量定义（科目列表、成绩等级）
│   ├── pages/            # 页面组件
│   │   ├── Login.jsx     # 登录页
│   │   ├── student/      # 学生端（成绩输入、结果、收藏、设置）
│   │   └── admin/        # 管理端（Dashboard、账号、规则、专业）
│   ├── styles/           # 全局样式
│   ├── utils/            # 工具函数
│   │   ├── api.js        # API接口（Mock）
│   │   └── mockData.js   # Mock数据
│   ├── App.jsx           # 路由配置
│   └── main.jsx          # 入口文件
├── index.html
├── package.json
└── vite.config.js
```

## 💡 技术栈

- **React 18** - UI框架
- **React Router 6** - 路由管理
- **Vite** - 构建工具
- **纯CSS** - 黑白简洁风格

```bash

npm run build

# 预览生产版本
npm run preview
```

## 📝 注意事项

1. 数据存储在浏览器LocalStorage中，清除浏览器数据会丢失收藏等信息
2. 权益点数在计算时会被扣除，刷新页面后重置为初始值
3. 专业推荐算法基于Mock数据，实际应用需要替换为真实算法
4. 所有账号密码存储在前端mockData.js中，仅供演示使用
